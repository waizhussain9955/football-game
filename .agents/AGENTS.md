# Engineering Rules for Project Striker

- Never rewrite completed systems.
- Never break public APIs.
- Every feature must compile.
- Every class must have documentation.
- Every subsystem must include unit-testable logic where applicable.
- Every manager must expose clear interfaces.
- Avoid circular dependencies.
- Avoid God Classes.
- Avoid Blueprint-only gameplay.
- Core gameplay must remain in C++.
- Blueprints are for editor integration and designer workflows only.
- Every milestone must produce a playable build.
